# A2 - Features and Public API

## Feature Set
1. `F-INGEST`: Data ingestion, timestamp harmonization, and quality controls.
2. `F-PROXIES`: Physics-guided proxy computation module.
3. `F-CLUSTER`: PAM-based operating-state reconstruction with medoid extraction.
4. `F-RISK`: Reduced-order susceptibility score and ordinal risk labeling.
5. `F-MODEL`: Interpretable cumulative-logit risk model fitting and inference.
6. `F-BENCH`: Benchmark comparator suite.
7. `F-VALIDATE`: Temporal CV, sensitivity, and robustness evaluators.
8. `F-ARTIFACT`: Manuscript table/figure generation.
9. `F-PIPELINE`: Interactive pipeline CLI wizard and batch workflow.

## Objective-to-Feature Mapping
- ACC-OBJ-01 -> F-INGEST, F-PROXIES
- ACC-OBJ-02 -> F-CLUSTER
- ACC-OBJ-03 -> F-RISK
- ACC-OBJ-04 -> F-MODEL
- ACC-OBJ-05 -> F-BENCH, F-VALIDATE, F-ARTIFACT

## Interactive Pipeline Requirement
- CLI entrypoint: `resonance-pipeline`.
- Behavior:
  - prompts for dataset path, configuration, output directory;
  - runs full workflow from ingest to artifact generation;
  - writes machine-readable logs and summary output.

## Non-Coder Friendly Surface
- Top-level short functions:
  - `read(path)` -> load dataset
  - `screen(path, output_dir)` -> one-command full workflow
- Method chaining:
  - `ScreeningSession().load(path).prep().fit().summary()`

## Non-functional API expectations
- All public functions return typed objects.
- User-facing errors include remediation tips.
- Deterministic behavior under fixed random seeds.
