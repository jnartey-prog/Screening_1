# A5 - Quality, CI, and Operations

## Quality Gates
- Lint: `ruff check .`
- Type checks: `mypy src`
- Tests: `pytest -q`
- Coverage target: minimum 85% line coverage for core modules.

## CI Requirements
- Workflow runs on push/PR:
  - install via `uv`
  - lint and type checks
  - unit/integration tests
  - pipeline smoke test with fixture data
  - artifact-generation smoke test

## Operational Requirements
- Structured logs stored at `logs/` (jsonl + text).
- Run logs include: `run_id`, `timestamp`, `stage`, `status`, `duration_ms`, `dataset_name`, `input_hash`, `config_hash`, `error_code`.
- Stats logs include: `run_id`, `timestamp`, `split_id`, `metric_name`, `metric_value`, `confidence_interval`, `sample_size`, `class_distribution`, `sensitivity_scenario`.
- Retention: 180 days run logs, 365 days stats logs.

## Release and Governance Readiness
- Generate: `TRACEABILITY.csv`, `LITERATURE.bib`, `REPRODUCIBILITY.md`, `preregistration.json`.
- No placeholder tokens permitted in A1-A5 or public APIs.
- Interactive pipeline remains mandatory for release acceptance.
